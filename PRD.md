## 1. 产品概述

### 1.1 一句话定位
**纳格（Nage）** 是一个**轻量、自托管**的物品收纳管理系统，让你随时知道「家里有什么」「放在哪里」「还有几个」。

### 1.2 解决什么问题
- 找不到东西（钥匙、充电器、文件……）
- 重复购买（忘了已经有的物品）
- 收纳空间混乱，不清楚柜子里到底有什么
- 借出去的东西忘了还 / 保质期过了不知道

### 1.3 目标用户（✅ 已确认：个人 / 家庭优先）
| 优先级 | 用户类型 | 典型场景 |
|---|---|---|
| **P0** | **个人** | 整理个人物品、电子配件、收藏 |
| **P0** | **家庭** | 全家共享一个"家庭仓库"（家 → 房间 → 柜子 → 抽屉），管理员给家庭成员开账号 |
| P2 | 小团队/工作室 | 暂不在 v1 范围 |

### 1.4 非目标（明确不做）
- 不做电商/交易
- 不做云端 SaaS 多租户（初期只做单实例自托管）
- 不做 AI 自动识别（v1 不含 OCR/图像识别，但保留扩展位）

## 2. 核心使用场景

1. **快速记录**：手机拍个照 + 输入名称 + 选个位置 → 物品入库
2. **快速查找**：输入"充电线" → 出现所有相关物品 + 位置 + 数量
3. **位置盘点**：进入"主卧衣柜"，看到里面所有物品列表
4. **二维码定位**：在收纳盒外贴一个二维码，手机扫码 → 直接看到盒内物品
5. **借出登记**（v2）：把工具借给同事，系统记录归还日期，到期提醒

## 5. 数据模型（SQLite）

```sql
-- 用户（管理员模式，无公开注册）
users (
  id, username, password_hash,
  nickname, avatar,
  role,                  -- 'admin' | 'member'
  is_active,             -- 管理员可停用某成员
  created_at, last_login_at
)

-- 空间（隔离数据用）
spaces (id, name, owner_id, created_at)

-- 位置（树形）
locations (
  id, space_id, parent_id NULL,
  name, description, cover_image,
  sort_order, created_at
)

-- 分类
categories (id, space_id, name, icon, color, sort_order)

-- 标签
tags (id, space_id, name, color)

-- 物品
items (
  id, space_id, name, description,
  category_id, location_id,
  quantity, unit,           -- 数量和单位（如"5 个"）
  value,                    -- 可选价值
  expired_at,               -- 可选过期时间
  created_at, updated_at
)

-- 物品图片
item_images (id, item_id, path, sort_order)

-- 物品-标签多对多
item_tags (item_id, tag_id)

-- 借出记录（v1.1）
borrow_records (id, item_id, borrower, borrowed_at, expected_return_at, returned_at, note)
```

## 6. 技术选型（✅ 已确认）

| 层 | 选型 | 备注 |
|---|---|---|
| 框架 | **Next.js 15（App Router）** | 前后端一体，Server Actions 处理写入，Route Handlers 暴露公开 API（如扫码接口） |
| 语言 | **TypeScript** | 全栈类型安全 |
| 样式 | **Tailwind CSS v4** | 移动优先响应式 |
| 组件 | **shadcn/ui**（基于 Radix UI） | 复制源码到项目，自由改；桌面/移动共用一套 |
| 图标 | **lucide-react** | shadcn 默认搭配 |
| 表单 | **react-hook-form + zod** | 校验 + 类型复用 |
| 数据库 | **SQLite**（单文件 `data/nage.db`） | 零配置，备份 = 复制文件 |
| ORM | **Drizzle ORM** | 轻量、SQL 友好、类型推导强；比 Prisma 启动快 |
| 鉴权 | **jose**（JWT）+ **bcryptjs** | 无 NextAuth 复杂度，自己掌控 |
| 状态 | React Server Components 为主，**zustand** 仅用于少量客户端状态 | 尽量少客户端 JS |
| 图片 | 本地 `public/uploads/`，**sharp** 压缩 | 服务端处理 |
| 二维码 | **`qrcode`**（生成 PNG） + **`jsqr`**（移动端扫码） | |
| 部署 | **Docker**（推荐） / 裸机 `pnpm start` | Nginx/Caddy 反代，HTTPS 走 Caddy 自动证书 |

- **移动优先**：先做移动端，再适配 PC

- **空白友好**：首次进入有引导示例数据
- **支持黑暗模式**：v1.1 加上

## 12. 鉴权设计补充（管理员模式）

### 12.1 账号生命周期
1. **首次部署**：从 `.env.local` 读 `ADMIN_USERNAME` + `ADMIN_PASSWORD`，启动时若 `users` 表为空则创建唯一管理员
2. **添加成员**：管理员登录 → "成员管理"页 → 输入用户名/昵称/初始密码 → 创建
3. **停用成员**：管理员可置 `is_active = false`，被停用账号登录返回 403
4. **改密**：用户自己在"我的 → 修改密码"页改；管理员可在成员管理页重置

### 12.2 权限
| 操作 | 管理员 | 普通成员 |
|---|---|---|
| 添加/删除成员 | ✅ | ❌ |
| 所有物品的增删改 | ✅ | ✅（自己的空间内） |
| 导入/导出 | ✅ | ❌（v1.0 仅管理员） |
| 修改系统设置 | ✅ | ❌ |

### 12.3 安全要点
- 密码 bcrypt cost=12 哈希
- JWT 存 httpOnly + Secure + SameSite=Lax cookie
- 登录失败计数存数据库（防绕过 cookie 锁定）
- 所有 Server Actions 内置 CSRF token
- 部署必须 HTTPS（否则 Secure cookie 失效）
